The rubric
Clarity
Can I tell what this is?
›
The very first question a new user asks. Before anything else works, the purpose has to be legible. It covers headline copy, hero layout, value proposition, CTA labeling. If this fails, nothing else matters.
Visual hierarchy
Where do I look?
›
Once the user understands the purpose, their eye needs guidance. This covers type scale, contrast, layout flow, and visual weight. It's the difference between a page that reads naturally and one where everything competes equally.
Trust signals
Should I believe this?
›
This is where the user's skepticism kicks in. Social proof, testimonials, press mentions, pricing transparency, security badges — anything that answers "why should I trust you?" It matters most on landing pages and checkout flows.
Transparency
Do I understand what's happening?
›
Distinct from trust — this is about system clarity rather than credibility. Does the user know what the product does with their data? Does AI-powered functionality explain itself? Is pricing honestly framed? This category has become especially important as AI features become common in products.
Accessibility
Can everyone use this?
›
The technical layer. Contrast ratios, touch target sizes, keyboard navigation, screen reader compatibility, WCAG AA compliance. Unlike the others, this one has objective pass/fail criteria — which is why the callouts here are the most specific.
Feedback loops
Does the UI talk back to me?
›
The final layer — how well does the interface communicate state? Loading, empty, error, and success states. A product can pass every other category and still feel broken if it goes silent during async operations or fails to explain what went wrong.
How it works
01
Give it a target
Paste a URL or drop in a screenshot. Let the system see exactly what users see.
02
AI runs the audit
The system analyzes the interface across six UX dimensions. Each category is scored 1–10 with a blurb on how to fix the errors.
03
Get specific fixes
Every issue includes developer-ready and design-ready callouts: contrast ratios, WCAG references, exact CSS fixes. Not vague advice.
